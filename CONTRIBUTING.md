# Contributing

Thank you for contributing to E.T Agent Website.

## Development Workflow

1. Confirm the repository builds and lints before making changes.
2. Keep changes focused on the approved phase and scope.
3. Avoid unrelated refactors or behavior changes.
4. Update documentation when repository conventions change.
5. Run validation before requesting review.

## Local Checks

```sh
npm run lint
npm run build
```

## Pull Request Expectations

- Describe the purpose and scope of the change.
- Note validation commands and their results.
- Call out any known limitations or follow-up work.
- Keep application behavior unchanged unless the phase explicitly approves it.

## Code of Conduct

All contributors are expected to follow the [Code of Conduct](./CODE_OF_CONDUCT.md).
