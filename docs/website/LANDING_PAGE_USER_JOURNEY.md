# E.T Agent Homepage User Journey

Status: W3A architecture specification; W3A.6 website scope revision  
Scope: Homepage journey, decision path and section-to-intent mapping for the official E.T Agent Website.

Historical note: the original planning centred around a Landing Page. As the project matured, the scope expanded into the official E.T Agent Website. The Homepage is now one module within the wider website, and this document preserves the frozen W3A Homepage journey.

## 1. Primary Audience

The homepage must serve enterprise buyers and evaluators who care about operational trust:

- Executives evaluating AI adoption risk.
- Enterprise architects assessing system fit.
- Compliance and governance teams reviewing accountability.
- Security stakeholders checking authority boundaries.
- Operations leaders considering AI-assisted execution.

These users are likely skeptical of generic AI claims. The journey must show control, policy, auditability and human accountability before making broader capability claims.

## 2. Journey Thesis

The user should move through this understanding:

1. Modern AI creates governance and trust challenges.
2. Governance is what transforms AI capability into accountable enterprise systems.
3. E.T Agent is governance-first enterprise AI.
4. Its core difference is separating intelligence from authority.
5. Execution follows a controlled workflow.
6. The platform has enterprise capabilities and architecture principles.
7. It applies to high-accountability industries.
8. Development is phased and transparent.
9. The next step is a serious architecture conversation.

## 4. Emotional Arc

The page should create confidence, not excitement for its own sake.

Initial state:

- Curious but cautious.
- Concerned about uncontrolled AI action.
- Tired of broad AI claims.

Desired final state:

- Clear on E.T Agent's governance-first category.
- Confident that the platform prioritizes control over autonomy.
- Ready to discuss architecture, risk and implementation needs.

## 5. Narrative Flow

### Act 1: Establish the Category

Sections:

- Hero
- Enterprise trust banner
- The Problem
- Why Governance Matters

Narrative job:

- Frame AI governance as the central issue.
- Avoid opening with model capability, speed or futuristic imagery.
- Make the user's enterprise concern feel understood.
- Explain why governance is the architectural answer to uncontrolled AI capability.

Key message:

> AI may reason quickly, but enterprise action requires controlled authority.

### Act 2: Define the Product and Mechanism

Sections:

- Introducing E.T Agent
- How E.T Agent Works
- Comparison

Narrative job:

- Define E.T Agent plainly.
- Make the workflow memorable.
- Show that policy and human approval are built into the system rather than added later.

Key message:

> E.T Agent separates intelligence from authority.

### Act 3: Prove Enterprise Fit

Sections:

- Core Capabilities
- Architecture Principles
- Industry Use Cases
- Development Roadmap

Narrative job:

- Translate the concept into enterprise capabilities.
- Show architecture discipline.
- Give enough detail for technical and governance stakeholders.

Key message:

> The platform is designed for accountable automation in environments where execution must be governed.

### Act 4: Convert

Section:

- Final CTA

Narrative job:

- Invite a practical next step.
- Keep the ask enterprise-appropriate.

Key message:

> Start with an architecture review, not a generic demo.

## 6. CTA Strategy

Primary CTA:

- `Request architecture review`

Use this in the Hero and Final CTA. It signals serious enterprise evaluation.

Secondary CTA options:

- `View governance model`
- `Contact the team`

Avoid:

- `Get started free`
- `Try AI now`
- `Build with agents`
- `Unlock the future`

## 7. Messaging Ladder

The implementation should maintain this message priority:

1. Governance-first AI.
2. Human-approved execution.
3. Policy before power.
4. Fail-closed execution.
5. Immutable auditability.
6. Deterministic workflows.
7. Enterprise trust.
8. Explainability and accountability.

When copy must be shortened for responsive layouts, preserve the top-ranked messages first.

## 8. Key Conversion Moments

Hero:

- User understands category and can immediately take the primary CTA.

How It Works:

- User sees the signature controlled workflow and understands why E.T Agent is different.

Architecture Principles:

- Technical stakeholders see seriousness and discipline.

Final CTA:

- User is invited into an architecture-led conversation.

## 9. Mobile Journey Requirements

Mobile users must receive the same argument in a more linear form:

- No horizontal-only workflow dependency.
- No hidden comparison content.
- CTA buttons should remain easy to tap.
- The workflow must read as a clean vertical timeline.
- Section headings must be scannable without relying on dense paragraphs.

## 10. Desktop Journey Requirements

Desktop layouts should use available width to create clear comparison and workflow comprehension:

- Hero can split narrative and workflow preview.
- Full workflow can use a horizontal or stepped rail.
- Comparison can use a native table.
- Capability and industry sections can use multi-column grids.
- Roadmap can use a horizontal phase view if it remains readable.

## 11. Success Criteria for Journey

The homepage succeeds if a first-time enterprise visitor can answer:

- What problem does E.T Agent address?
- What is E.T Agent?
- Why is separating intelligence from authority important?
- What is the controlled execution workflow?
- What enterprise controls are available?
- What kinds of organizations is this for?
- What is the next step?
